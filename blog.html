    Reflection on the readings I selected, relating it to my experience with internet today:

    The readings I engaged with were: What Hypertext is by Noah Wardrip-Fruin and As we may think by Vannevar Bush. Regarding What Hypertext is, I grasped a basic understanding of what hypertext is using a definition by Marie Laure Ryan, “In hypertext… the reader determines the unfolding of the text by clicking on certain areas, the so-called hyperlinks, that bring to the screen other segments of text.” This definition got me to understand how interactivity deals with links, how one component is connected to another component, for example how the link to a published scholarly essay will direct you to a website that contains the PDF version of the essay where users can view and read the text. The reading also got me to learn about hypermedia which are branches and presentations which respond to user actions and how they are systems of prearranged words and pictures, which can be freely explored.

    The thing that caught my attention in As we may think, was how the text focused on the anticipation and vision of the technological devices we have today. It made me understand the limitations thatpeople had before these advancements, for example how people had to store information in books, in organized and systematic ways that were time consuming and inefficient compared to the rapid technology we share today that allows users to efficiently locate information from vast sets of data. Instead of going to the library, we can access a digital library through a website such as Google Scholar, to be able to read the same information that is present in physical books. The reading also allowed me to understand the concept of Memex, although I don’t have a solid understanding of the concept, I understand how it is essentially a private file or library for individual use, it allows users to store various materials such as records and personal notes and offers them easy access to it. I think of Memex as a house for the user, where they can place their information in different rooms of the house and arrange them in any way they want.

    Reflect on interaction and the www: what does interaction mean to you?

    Let me start of by mentioning my favourite part about the lecture: “we are in the business of manipulation”. I never thought of how as web designers, we manipulate the user to make certain actions such as clicking on a link on our websites, it made me think of the psychological part of it, and how we should understand our user to ultimately manipulate them.

    Interaction to me is how input leads to output. In order for something to be interactable there has to be an outcome as a result of an input. In web design, I think the interaction can be seen when the user clickson a user interface element (home button) and it directs the user to the respective page (home page).

    The lecture taught me more about interactivity and how it should be responsive and not respond in a completely random fashion. I don’t think I’ve ever thought of how websites are not random, how when you click on the close button, it closes the tab and not shut down the computer.

    I also feel like a huge part of interactivity is ‘feedback’, because feedback allows the user to know that something has happened from something they did, an example of this would be how the light switches on once the laptop charger is switched on.

    Week 3: Blogpost

    User Alignment:

    User alignment refers to making sure a website is easy to use and clearly shows what it's meant for. It's about ensuring that the website's goals match what users want, making everything simple and straightforward. It involves ensuring that the design and interface are seamlessly aligned with the user’s expectations, making the overall experience straightforward and user-friendly. Essentially, it's like making sure the website and the user are on the same page and understand each other easily. The target audience makes it easier to consider what the user would expect from the page, for example if the target audience was fitness and health enthusiasts the website would probably have content centred around fitness, health recipes, wellness tips and motivation. It would also feature workout videos, tips and meal-planning tools.

    What about my website?

    My website would be used to showcase and promote my portfolio and my target audience will be potential employers or potential collaborators for passion projects. I want my website to be easy to use by including the necessary input controls such as links, this ensures that navigating through my portfolio is straightforward, enabling visitors, especially my target audience to easily access and explore my showcased work. The intention with regards to the user is to create a seamless and intuitive experience that effectively communicates my skills and expertise to users especially those with my shared interests.

    User Experience:

    This refers to what users feel when they interact with the website. I would start off with the information structure and determine what content goes with another and where it will be positioned. Then determine the options the user has at each point, if it’s the page for my blogs, the user should be able to access any blog post at any time. From the lecture I learnt how user experience considers many types of users, for example users with bad connection speeds, will the website offer text to represent the images? Another example would be users who are colour blind, will the website has a feature that is capable of adjusting colours to be user-friendly for those who are colour blind. Another thing I would want my website to have is feedback, to help the user know what results are caused by their input, for example interactive elements like buttons changing colour to acknowledge user interactions, providing visual feedback. Another example would be load spinners or progress bars, when my website is processing a request or loading images, the spinners will represent the action of loading which gives the user feedback about that process.
